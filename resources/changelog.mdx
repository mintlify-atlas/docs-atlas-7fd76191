- Extracted shared signal collector test helper

- Canceled timed-out completion waiters in AgentServer
- Preserved file read errors during append
- Returned normalized errors for corrupt entry logs
- Avoided atom creation from persisted journal kinds
- Restored agent metadata export for discovery
- Returned routing error struct for unknown signals
- Normalized refinement errors for Zoi validation in plugins
- Made ETS `append_thread` atomic and revision-safe
- Sent `jido.agent.stop` when stopping child
- Normalized id in both constructor paths to guarantee UUID7
- Fixed silent cron tick failure
- Normalized util and storage-facing error contracts
- Offloaded runtime dispatch from sensor process

- Bumped `jido_action` and `jido_signal` to 2.0.0-rc.5

## 2.0.0 Architecture Preview - 2025-12-31

### Added
- **Instance-Scoped Supervisors**: User-owned, instance-scoped supervisors instead of global singletons
- **Jido Supervisor Module**: Manages per-instance Registry, TaskSupervisor, and AgentSupervisor
- **Agent Lifecycle API**: Functions for `start_agent/3`, `stop_agent/2`, `whereis/2`, `list_agents/1`
- **Test Isolation**: `JidoTest.Case` module for automatic test isolation
- **Per-Instance Scheduler**: Scoped scheduler support
- **Multi-Agent Coordination**: `await/3` and `await_child/4` for agent completion
- **Discovery Delegates**: Discovery functions via main Jido module

### Breaking Changes
- **Explicit Supervision**: Must add `{Jido, name: MyApp.Jido}` to supervision tree
- **Agent Start**: Use `Jido.start_agent(instance, agent, opts)` instead of `AgentServer.start/1`
- **Instance Option**: Pass `jido: MyApp.Jido` when starting agents directly

### Changed
- Refactored from global singletons to instance-scoped architecture
- Agents register in instance-specific Registry
- Per-instance TaskSupervisor and AgentSupervisor
- Updated all examples to use new 2.0 patterns
