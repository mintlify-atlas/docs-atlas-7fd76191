## Overview

A Sensor is a pure behaviour that transforms external events into Signals. Sensors are stateless modules that define how to initialize, handle events, and emit signals. The runtime execution is handled by a separate SensorServer.

## Configuration Options

<ParamField path="name" type="string" required>
  The sensor name. Must contain only letters, numbers, and underscores.
</ParamField>

<ParamField path="description" type="string">
  Description of what the sensor does.
</ParamField>

<ParamField path="schema" type="Zoi.schema()">
  Zoi schema for config validation.
</ParamField>

## Callbacks

### init/2

Initialize the sensor state. Called when the sensor is started.

<ParamField path="config" type="map" required>
  Validated configuration map (parsed against the sensor's schema).
</ParamField>

<ParamField path="context" type="map" required>
  Runtime context (e.g., sensor id, parent process info).
</ParamField>

<ResponseField name="return" type="{:ok, state} | {:ok, state, directives} | {:error, reason}">
  Initial state for the sensor, optionally with startup directives.
</ResponseField>

```elixir
def init(config, _context) do
  state = %{metric: config.metric, last_value: nil}
  {:ok, state, [{:schedule, 1000}]}
end
```

### handle_event/2

Handle an incoming event and produce directives.

<ParamField path="event" type="any" required>
  The incoming event (format depends on the source).
</ParamField>

<ParamField path="state" type="any" required>
  Current sensor state.
</ParamField>

<ResponseField name="return" type="{:ok, state} | {:ok, state, directives} | {:error, reason}">
  Updated state with optional directives to execute.
</ResponseField>

**Directives:**
- `{:emit, signal}` - Emit a signal to the connected agent
- `{:schedule, interval_ms}` - Schedule a `:tick` event after interval
- `{:schedule, interval_ms, event}` - Schedule a custom event after interval
- `{:connect, adapter}` - Connect to an external source
- `{:connect, adapter, opts}` - Connect with options
- `{:disconnect, adapter}` - Disconnect from a source
- `{:subscribe, topic}` - Subscribe to a topic/pattern
- `{:unsubscribe, topic}` - Unsubscribe from a topic

```elixir
def handle_event(:tick, state) do
  signal = Jido.Signal.new!(%{
    source: "/sensor/example",
    type: "example.tick"
  })
  {:ok, state, [{:emit, signal}, {:schedule, 1000}]}
end
```

### terminate/2

Called when the sensor is shutting down. Default implementation returns `:ok`.

<ParamField path="reason" type="any" required>
  The shutdown reason.
</ParamField>

<ParamField path="state" type="any" required>
  Current sensor state.
</ParamField>

<ResponseField name="return" type=":ok">
  Shutdown complete.
</ResponseField>

```elixir
def terminate(_reason, state) do
  cleanup_resources(state)
  :ok
end
```

## Example Sensor

```elixir
defmodule MySensor do
  use Jido.Sensor,
    name: "my_sensor",
    description: "Monitors a specific metric",
    schema: Zoi.object(%{
      metric: Zoi.string()
    })

  @impl true
  def init(config, _context) do
    {:ok, %{metric: config.metric, last_value: nil}}
  end

  @impl true
  def handle_event({:metric_update, value}, state) do
    signal = Jido.Signal.new!(%{
      source: "my_sensor",
      type: "metric.updated",
      data: %{value: value, previous: state.last_value}
    })
    {:ok, %{state | last_value: value}, [{:emit, signal}]}
  end
end
```

## Polling Sensor

```elixir
defmodule PollingSensor do
  use Jido.Sensor,
    name: "polling_sensor",
    description: "Polls an external API",
    schema: Zoi.object(%{
      url: Zoi.string(),
      interval_ms: Zoi.integer() |> Zoi.default(5000)
    })

  @impl true
  def init(config, _context) do
    state = %{url: config.url, interval: config.interval_ms}
    # Start polling immediately
    {:ok, state, [{:schedule, config.interval_ms, :poll}]}
  end

  @impl true
  def handle_event(:poll, state) do
    case fetch_data(state.url) do
      {:ok, data} ->
        signal = Jido.Signal.new!(%{
          source: "/sensor/polling",
          type: "data.received",
          data: data
        })
        {:ok, state, [{:emit, signal}, {:schedule, state.interval, :poll}]}

      {:error, reason} ->
        {:ok, state, [{:schedule, state.interval, :poll}]}
    end
  end

  defp fetch_data(url) do
    # Implement API call
  end
end
```

## PubSub Sensor

```elixir
defmodule PubSubSensor do
  use Jido.Sensor,
    name: "pubsub_sensor",
    description: "Listens to Phoenix PubSub",
    schema: Zoi.object(%{
      topic: Zoi.string()
    })

  @impl true
  def init(config, _context) do
    state = %{topic: config.topic}
    {:ok, state, [{:subscribe, config.topic}]}
  end

  @impl true
  def handle_event({:pubsub_message, payload}, state) do
    signal = Jido.Signal.new!(%{
      source: "/sensor/pubsub",
      type: "message.received",
      data: payload
    })
    {:ok, state, [{:emit, signal}]}
  end

  @impl true
  def terminate(_reason, state) do
    # Cleanup handled by SensorServer
    :ok
  end
end
```

## Type Definitions

```elixir
@type sensor_directive ::
  {:schedule, pos_integer()}
  | {:schedule, pos_integer(), term()}
  | {:connect, atom()}
  | {:connect, atom(), keyword()}
  | {:disconnect, atom()}
  | {:subscribe, term()}
  | {:unsubscribe, term()}
  | {:emit, Jido.Signal.t()}
```
